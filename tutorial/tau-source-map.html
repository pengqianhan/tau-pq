01 / Layer Boundary

先记住一个方向：tau_coding → tau_agent → tau_ai

Tau 的关键不是“代码很多”，而是边界清楚：模型适配在底层，portable brain 在中间， CLI/TUI/resources/session 都在应用层。核心 brain 不知道 Textual、Rich、文件路径或 slash commands。

Ctrl/Cmd + 滚轮缩放；拖拽平移；双击适配；右上角按钮可展开。

Ready
model layer

tau_ai

把外部模型 API 翻译成 Tau 的 provider-neutral stream。上层只看 ProviderEvent，不看 vendor payload。

  • provider.py 定义 ModelProvider 协议
  • events.py 定义 start / delta / thinking / tool / end / error
  • fake.py 给 loop tests 回放确定事件
  • openai_compatible.py 适配 chat completions stream
portable brain

tau_agent

真正可复用的 agent brain：消息、工具、事件、纯循环、harness、append-only session primitives。

  • loop.py 运行“模型响应 → 工具结果 → 继续”的循环
  • harness.py 管 transcript、队列、取消、listener
  • messages.py 三类 transcript message
  • session/ JSONL 友好的历史模型
coding app

tau_coding

把 brain 包成能工作的编码 Agent：CLI、Textual TUI、工具、resources、provider config、sessions、export、compaction。

  • cli.py 入口：print mode / TUI / sessions / export
  • session.py app shell 的中心
  • tools.py read / write / edit / bash
  • tui/ Textual UI，只消费事件
02 / Event Contract

事件是 Tau 的公共接口

Provider chunk 先被 tau_ai 变成 provider events；run_agent_loop 再把它们变成 agent events。print renderer 和 Textual adapter 都只消费这些事件。

这张图只画主路径；session branch、login、export 等控制流在应用层处理。

Ready
core cycle

run_agent_loop 做什么

每 turn 调 provider、发消息事件、收 assistant message、执行工具、把 ToolResultMessage 放回 transcript。没有工具调用时，检查 steering/follow-up 队列后停止。

stateful wrapper

AgentHarness 做什么

管 transcript、运行互斥、取消 token、event listeners、queued steering/follow-up，以及被中断工具调用的 transcript 修复。

03 / App Shell

tau_coding 是产品复杂度所在

tau_agent 故意小；真正多的代码在 app shell：配置、登录、资源发现、TUI、会话树、导出、 自动 compaction。这些都不应该倒灌进 portable brain。

源码大小不是依赖方向

tau_coding 约 16k 行，远大于 tau_agent 的约 1.3k 行。 这正说明产品外壳承载复杂度，而可复用 brain 仍保持小而稳定。

最重的应用模块

TUI 和 session 是主要重量级区域；如果要理解全系统，先不要从 tui/app.py 开始，应该先读 core types、loop、harness，再回到 app shell。

05 / Reading Route

推荐阅读顺序：先 brain，再 shell，最后 TUI

如果你从最大文件开始，会被产品细节淹没。下面这条路线按抽象层级走，并和现有 tutorial/README.md 的章节互补。

00

先读现有中文教程的第 0 章

tutorial/README.md 已经解释三层心智模型、dev-notes 和 tests 的关系。

01

读最小数据结构

tau_agent/messages.py、tools.py、events.py，再跑 tests/test_agent_types.py。

02

读 provider stream

tau_ai/provider.py、events.py、fake.py，再看 tests/test_tau_ai.py 里的 streaming 断言。

03

读 agent loop 和 harness

tau_agent/loop.py 解释 agent 为什么能用工具；harness.py 解释 transcript、queue 和 cancel。

04

读 coding tools

tau_coding/tools.py 把本地文件和 shell 行为隔离在 app 层，产出 provider-neutral AgentToolResult。

05

读 CodingSession 拼装

tau_coding/session.py 是 app shell 的中心。带着“它如何喂 harness”这个问题读。

06

最后读 CLI / rendering / TUI

cli.py 分流 print/TUI；rendering/ 和 tui/adapter.py 证明 frontend 只消费 AgentEvent。
